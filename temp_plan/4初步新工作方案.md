# Codex 项目工作文档：博客网站开发计划

之前ZBlog的工作不再继续D:\MyCode\ZBlogProject\ZBlog，

现在我们有新的工作目标

## 1. 项目目标

我要开发一个**生产级个人品牌博客网站**。

当前阶段的核心目标不是 DevWiki，也不是做一个简单 Demo，而是先把一个完整、现代、可上线的博客网站做出来。

长期方向可以规划为：

```text
个人品牌首页
+ 技术博客
+ 项目展示
+ 后台管理
+ 内容运营
+ 后续扩展 
```

但当前重点只有一个：

```text
先把博客网站做好。
```

未来参考目标类似：

```text
https://bugstack.cn/
```

也就是：
不是普通博客皮肤，而是一个具备个人品牌感、技术内容沉淀能力、后台管理能力、长期运营能力的个人站点。

------

## 2. 参考项目

当前重点参考项目：

```text
https://github.com/talen8/FlecBlog
```

我看重 FlecBlog 的原因是：

1. 它是一个比较新的开源博客项目，也比较完善，具备之前参考项目的综合优点；
2. 仍在持续迭代，我也想做成一个具备个人技术成长的长期价值的项目；
3. 比很多几年不更新的博客项目更值得学习；
4. 它包含前台博客、后台管理、服务端能力；
5. 它的功能完整度、页面效果和工程组织值得参考。

但是注意：

```text
FlecBlog 后端是 Go。
我的项目后端必须使用 Java。
```

所以本项目不是简单 fork FlecBlog，也不是把 Go 后端机械翻译成 Java。

正确策略是：

```text
参考 FlecBlog 的产品功能、页面结构、后台能力、交互体验和工程思路；
前端可以：复用代码+局部重构/风格替换+新功能（比如我喜欢科技风，但是可能是先复用再改造比较好）
然后使用 Java 重新实现后端
```

## 7. 技术要求

后端使用 Java 技术栈。

之前讨论过的技术能力需要纳入项目规划，不要人为降低复杂度。

后端建议使用：

```text
Java 21
Spring Boot
Spring Security
JWT
MyBatis-Plus / MyBatis
Redis
MQ
es
canal/maxwell
虚拟线程
DDD 分层思想
MySQL / PostgreSQL
Docker Compose
OpenAPI / Swagger
定时任务
文件上传
缓存设计
异步事件
基础监控日志
```

说明：

```text
这些技术不是为了堆技术，而是要合理落到博客系统的真实业务场景中。
```

例如：

| 技术             | 落地场景                                     |
| ---------------- | -------------------------------------------- |
| Redis            | 文章阅读量、热门文章、登录状态、验证码、缓存 |
| MQ               | 评论通知、文章发布事件、统计事件、异步任务   |
| Java 21 虚拟线程 | 后台批处理、外部资源访问、IO 密集型任务      |
| DDD              | 文章、评论、用户、站点配置等模块边界         |
| 定时任务         | sitemap 生成、统计汇总、缓存刷新             |
| 文件上传         | 文章封面、Markdown 图片、站点资源            |
| OpenAPI          | 前后端接口协作                               |
| Docker Compose   | 本地开发和部署环境                           |
