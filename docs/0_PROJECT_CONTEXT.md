# 0_PROJECT_CONTEXT.md

> 本文职责：说明 ZBlog 的项目背景、当前目标、阶段边界、参考来源和关键取舍。

## 项目背景

ZBlog 不是简单复刻旧项目，也不是练手 Demo。它是一次重新启动的博客项目，前期积累过一些高价值资产，但过去的推进过程中出现了目标摇摆、前后端契约不一致、阶段边界不清等问题。

## 当前目标

当前阶段只做一件事：

```text
先把一个完整、现代、可上线的博客网站做出来。
```

第一阶段不把重心放在个人品牌官网、项目集、DevWiki Studio、相册、说说扩展等内容上。这些可以进入 roadmap，但不应影响当前主线。

## 参考原则

核心参考项目：

```text
https://github.com/talen8/FlecBlog
```

### 为什么重视 FlecBlog

- 项目新，而且还在持续迭代；
- 三端分离：博客前台、管理后台、后端服务；
- 博客功能相对完整；
- 前端审美和内容容器感值得复用；
- 部署和文档思路更完整。

## 技术取舍

### 前端

- 优先复用 FlecBlog 可直接使用的前端源码；
- 先迁入能跑通的博客站和管理端，再做局部改造；
- 不为了“原创”重新造一套已经很好用的页面。

### 后端

- 使用 Java 21 + Spring Boot 重新实现；
- 不机械翻译 FlecBlog 的 Go 后端；
- 参考它的数据模型、接口语义和业务闭环；
- 继续沿用 ZBlog 中已验证过的 Markdown 真源、HTML 快照、TOC、安全渲染等能力。

## 当前成功标准

- 本地可以启动前台、后台、Java 后端和依赖服务；
- 后台可以登录并管理文章；
- 前台可以浏览首页、列表、详情、分类、标签、归档、评论、友链；
- 文章 Markdown、代码块、图片、目录和 SEO 表现正常；
- 可以部署到云服务器并通过域名访问；
- 后续任务能按本文档持续推进，而不是每次重新解释目标。
