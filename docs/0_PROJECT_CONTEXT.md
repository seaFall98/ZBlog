# 0_PROJECT_CONTEXT.md

> 本文职责：说明 ZBlog2 的项目背景、当前目标、阶段边界、参考来源和关键取舍。

## 项目背景

之前的 ZBlog 工作不继续按原路线推进。旧项目里有高价值资产，但整体过程存在目标摇摆、前端路线反复、设计规格不足、任务边界不清等问题。

ZBlog2 是一次项目重启，不是简单复制旧项目，也不是做练手 Demo。

## 当前目标

当前阶段只做一件事：

```text
先把一个完整、现代、可上线的博客网站做好。
```

第一阶段不把重心放到个人品牌官网、项目集、DevWiki Studio、相册、说说等扩展线上。它们可以进入 roadmap，但不影响当前博客主线。

## 参考定位

核心参考项目：

```text
https://github.com/talen8/FlecBlog
```

看重 FlecBlog 的原因：

- 项目新，仍在持续迭代；
- 三端分离：博客前台、管理后台、后端服务；
- 博客功能较完整；
- 前端审美和内容容器感值得学习；
- 部署和文档思路较完整。

## 关键技术取舍

前端：

- FlecBlog 前端源码可复用就先用；
- 先迁入能跑通的博客端和管理端，再局部改造；
- 不为了“原创”重写已有好用页面。

后端：

- 使用 Java 21 + Spring Boot 重新实现；
- 不机械翻译 FlecBlog 的 Go 后端；
- 参考 FlecBlog 的数据模型、接口语义和业务闭环；
- 沿用旧 ZBlog 中已验证的 Markdown 真源、HTML 快照、TOC、安全渲染等能力。

## 当前成功标准

- 本地可以启动前台、后台、Java 后端和依赖服务；
- 后台可以登录并管理文章；
- 前台可以浏览首页、列表、详情、分类、标签、归档、评论、友链；
- 文章 Markdown、代码块、图片、目录和 SEO 表现正常；
- 可以部署到云服务器并通过域名访问；
- 后续任务能按文档持续推进，而不是每次重新解释目标。
