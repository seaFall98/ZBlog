# PRODUCT_PLAN.md

> 本文职责：定义当前博客网站的产品形态、核心用户、第一阶段范围和后续扩展边界。

## 产品定义

ZBlog2 当前是一个生产级个人博客网站：

```text
博客前台 + 内容管理后台 + Java 后端 + 可部署上线
```

它不是普通皮肤 Demo，也不是 Java CRUD 练习项目。

## 当前用户

- 站长：写文章、管理分类标签、处理评论、维护友链和站点配置；
- 访客：阅读文章、浏览分类标签归档、搜索内容、评论互动、查看友链和关于页；
- 后续用户：关注个人品牌、项目展示或 DevWiki 的访问者，但不作为当前核心。

## 第一阶段范围

必须完成：

- 博客首页；
- 文章列表；
- 文章详情；
- 分类、标签、归档；
- 搜索；
- 评论；
- 友链；
- 关于页；
- 后台登录；
- 后台文章管理；
- Markdown 编辑器；
- 分类标签管理；
- 评论和友链管理；
- 文件/图片管理；
- 站点配置；
- 基础统计；
- SEO、sitemap、feed；
- Docker 部署。

## 非当前主线

以下进入 roadmap，不在第一阶段强做：

- 像 `bugstack.cn` 一样强个人品牌首页；
- 项目集/作品展示；
- DevWiki Studio 产品页；
- 相册、说说；
- AI 摘要、MCP、复杂通知集成；
- 一键安装器。

## 产品验收标准

- 访客能稳定阅读和检索博客内容；
- 站长能通过后台完成日常内容管理；
- 前台视觉干净、完整、耐看，不粗糙；
- 后台不是临时表单堆叠，而是正式 CMS；
- 项目可以部署到云服务器并绑定域名访问。
