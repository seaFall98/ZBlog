# CODEX_WORKFLOW.md

> 本文职责：定义后续 Codex 在 ZBlog2 中如何按科学 AI coding 工作流持续推进。

## 每次任务开始

1. 读取 `README.md`、`AGENTS.md` 和 `docs/`；
2. 确认当前 phase；
3. 检查是否涉及 FlecBlog 前端源码；
4. 检查是否涉及旧 ZBlog 可迁移资产；
5. 给出本次任务目标和验证命令。

## 执行规则

- 小步推进，不一次性大改全站；
- 前端先迁入可运行源码，再局部修改；
- 后端先搭基础能力，再接业务模块；
- 任何迁移都要说明来源文件、目标文件和修改理由；
- 代码变更后同步更新 docs；
- 不把后续规划塞进当前 phase。

## 前端任务模板

```text
目标：
来源：_reference\FlecBlog\blog 或 _reference\FlecBlog\admin
迁入文件：
局部修改：
接口适配：
验收命令：
风险：
```

## 后端任务模板

```text
目标：
模块：
接口：
数据表：
测试：
验收命令：
风险：
```

## 需要向用户索取的内容

进入部署或视觉替换前，需要用户准备：

- 域名；
- 云服务器；
- 备案/HTTPS 策略；
- 站点名；
- Logo 或头像；
- 作者昵称和简介；
- GitHub、邮箱、社交链接；
- 初始文章；
- 文件存储选择：本地、对象存储或云厂商 OSS。

## 完成声明规则

只有在实际运行验证命令并看到成功输出后，才能说任务完成。
