# Batch 9 人工验收步骤

## 1. 文章阅读去重

1. 打开一篇已发布文章，记录阅读量。
2. 刷新同一篇文章。
3. 预期：10 秒防抖窗口内，同一浏览器/访客不会重复增加阅读量。
4. 等待 10 秒后再次进入或刷新，预期可以再次正常计数。

## 2. 热门文章双榜

1. 访问多篇文章，让其中一篇产生有效阅读。
2. 打开前台热门文章卡片，切换 `热门榜 / 总榜`。
3. 预期：`热门榜` 按 Redis 近期热度排序，并显示热度；`总榜` 按总阅读量排序，并显示阅读数。
4. 也可以分别调用 `/api/v1/articles/hot?type=recent&limit=10` 和 `/api/v1/articles/hot?type=total&limit=10` 验证。

## 3. 访问采集限流

1. 快速连续请求 `/api/v1/collect`。
2. 预期：超过阈值后返回明确 `429`，不是静默成功。
3. 正常访问频率不应影响浏览博客。

## 4. 站点统计缓存

1. 打开前台首页/侧边栏站点统计。
2. 预期：统计仍能正常显示。
3. Redis 重启或缓存失效后，统计能从 PostgreSQL 回退恢复。

## 5. Outbox + RabbitMQ

1. 在后台发布一篇草稿文章。
2. 查看后台通知或 outbox 列表。
3. 预期：发布文章生成 `ARTICLE_PUBLISHED` outbox 事件，并在消费后创建一条后台通知。
4. 重复触发 outbox 处理，预期不会重复创建同一事件的通知。
5. 停掉 RabbitMQ 后发布文章，预期业务不崩，outbox 记录保持 pending/failed；恢复 RabbitMQ 后可重试。

## 6. 后续规划确认

Redis+Nginx+Lua 本批不验收代码，只确认已进入后续部署/网关路线图。
