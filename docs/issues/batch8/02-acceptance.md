# Batch 8 人工验收清单

请先确认本地 Docker 栈已经运行，然后按下面步骤验收。

访问地址：

- 前台博客：`http://localhost:3000`
- 后台管理：`http://localhost:4000`
- 后端 API：`http://localhost:8080`

## 1. 前台文章阅读量

建议使用未登录窗口或浏览器无痕窗口测试。当前前台 tracker 保留了“超级管理员访问不计入统计”的既有逻辑。

1. 打开前台博客 `http://localhost:3000`。
2. 进入任意一篇已发布文章，记录当前阅读量。
3. 等页面完全加载后，刷新文章页，或重新打开这篇文章。
4. 确认阅读量会增加，不再一直停留在 `0`，并且当前前台显示值不应比后台稳定少 `1`。
5. 打开后台文章列表。
6. 确认同一篇文章在后台列表中的阅读量与前台表现一致，或至少能看到同一计数在持续增长。

## 2. 站点 PV 统计语义

1. 记录前台侧边栏或后台 dashboard 中的站点 PV。
2. 打开或刷新一篇文章一次。
3. 确认总 PV 没有出现明显的重复跳涨。
4. 预期语义：站点 PV 来自 `visit_events`，文章阅读量来自 `articles.view_count`，两者不再互相叠加造成重复统计。

## 3. 后台访问日志筛选

1. 进入后台访问日志页面。
2. 分别尝试按 URL/关键词、IP、访客 ID、浏览器、操作系统、时间范围筛选。
3. 确认表格结果会随筛选条件真实变化。
4. 确认浏览器和操作系统能显示类似 `Chrome`、`Safari`、`Windows`、`macOS` 这样的解析结果。
5. 地理位置如果显示 `unsupported` 是当前预期行为，不算失败。

## 4. 后台文章列表筛选

1. 进入后台文章列表。
2. 分别尝试关键词、发布状态、分类、标签、置顶、精选、过时、地点、时间范围筛选。
3. 确认表格结果会随筛选条件真实变化。
4. 特别确认发布状态、置顶、精选、过时这些布尔筛选不是只在前端假过滤。

## 5. 后台评论列表筛选

1. 进入后台评论列表。
2. 分别尝试关键词、显示状态、删除状态、父评论/子评论、时间范围筛选。
3. 确认表格结果会随筛选条件真实变化。

## 6. 后台文件列表筛选

1. 进入后台文件列表。
2. 分别尝试关键词、文件类型、状态、上传用途、文件大小范围、时间范围筛选。
3. 确认表格结果会随筛选条件真实变化。

## 7. 前台评论隐私边界

1. 打开一篇带评论的文章。
2. 查看前台评论区。
3. 确认评论卡片只展示评论者身份、评论时间、评论内容等必要信息。
4. 确认前台不展示评论者地区、IP、浏览器、操作系统等隐私相关字段。

## 8. 上传设置页诚实性

1. 进入后台设置里的上传设置页。
2. 确认页面只展示“本地存储”作为当前真实生效的存储方式。
3. 确认页面明确说明当前后端上传固定使用本地 `/uploads/**`。
4. 确认页面明确说明当前后端文件大小限制固定为 10MB。
5. 确认 S3、OSS、COS、R2、MinIO 等尚未实现的云存储入口没有被展示成“已支持”。

## 9. 导入导出入口诚实性

1. 进入后台设置里的导入导出页。
2. 打开文章导入弹窗。
3. 确认页面说明 Markdown 图片目前只支持已经上传的 `/uploads/**` 路径。
4. 确认远程图片、相对路径图片不会被宣传成“自动下载并上传”。
5. 在后台文章列表打开导出弹窗。
6. 确认 HTML 导出被描述为基础 HTML 导出，不声称完整公众号排版能力。
7. 确认 Markdown ZIP 导出说明只打包 `/uploads/**` 中已落盘的本地图片资源。

## 验收规则

如果以上验收通过，再通知 Codex 继续写 `05-final-report.md`、提交并推送 Batch 8。

如果验收失败，请记录失败现象。后续需要先写 `03-issues.md` 和 `04-fix-report.md`，修复后再重新验收。
