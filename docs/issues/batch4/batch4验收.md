下面按“本地栈启动 → 准备 RSS 源 → 正常刷新 → 持久化/已读/去重 → 失败场景”的顺序验收。

  1. 启动本地栈

  在项目根目录运行：

  docker compose up --build -d
  docker compose ps

  确认 server / admin / blog / postgres 都是 running / healthy。

  如果想看后端日志：

  docker compose logs -f server

  打开后台：

  http://localhost:4000

  用本地管理员账号登录。默认如果没改过，一般是：

  admin / admin123456

---
  2. 准备一个可用 RSS 源

  进入后台：

  友链管理 / Friends

  新增或编辑一个友链，重点填：

  名称：Batch4 RSS 验收源
  网站地址：随便填该站主页
  RSS 地址：填	一个真实可访问的 RSS / Atom 地址
  状态：通过 / 非待审核

  验收用 RSS 地址可以用你自己熟悉的博客 RSS，或者任意确认能打开的 RSS/Atom 地址。

  保存后，先不要急着验收 RSS 页面，确认这个友链不是“待审核”状态，因为本轮刷新只抓取：

  rss_url 不为空
  并且 is_pending = false

  的友链源。

---
  3. 正常刷新 RSS

  进入后台：

  RSS订阅

  点击顶部按钮：

  刷新RSS

  期望结果：

  出现成功提示，例如：RSS刷新完成：新增 N 篇

  然后看 RSS 列表。

  期望看到：

  1. 列表出现刚才 RSS 源里的真实文章。
  2. 来源列显示对应友链名称。
  3. 标题链接能跳到原文章地址。
  4. 发布时间/抓取时间有值。
  5. 新抓取文章默认是“未读”。

  如果提示：

  新增 0 篇

  不一定是失败，可能是这个 RSS 源之前已经抓过，或者源里没有新 item。可以换一个新的 RSS 源再试。

---
  4. 验证刷新后数据持久化

  在 RSS 页面按浏览器刷新，或者离开页面后重新进入：

  RSS订阅

  期望结果：

  刚才抓取到的 RSS 文章仍然存在。

  这一步证明不是前端临时数据，而是已经入库。

---
  5. 验证已读状态持久化

  找一篇“未读”的 RSS 文章，点击：

  标记已读

  期望结果：

  状态从“未读”变成“已读”。
  未读角标数量减少。

  然后刷新页面，或者退出 RSS 页面再进入。

  期望结果：

  这篇文章仍然是“已读”。

  这一步证明 read/unread 是数据库状态，不是前端临时状态。

---
  6. 验证重复刷新不会重复插入

  在同一个 RSS 源不变的情况下，再次点击：

  刷新RSS

  期望结果：

  如果源没有新文章，提示新增 0 篇。
  RSS 列表里不会出现同一标题/同一链接的重复文章。
  刚才标记为已读的文章仍然保持已读。

  重点看两件事：

  1. 不重复插入同一个 RSS item。
  2. 已读状态没有被刷新覆盖回未读。

---
  7. 验证失败 RSS 源

  回到友链管理，新增或编辑一个友链，填一个明确错误的 RSS 地址，例如：

  名称：Batch4 Broken RSS 验收源
  网站地址：https://example.com
  RSS 地址：https://example.com/not-rss.xml
  状态：通过 / 非待审核

  保存后回到：

  RSS订阅

  点击：

  刷新RSS

  期望结果：

  页面提示类似：RSS刷新完成：新增 X 篇，失败 1 个源

  这说明失败不是静默成功。

  本轮前端只展示聚合失败数量；具体失败原因已经写入后端 friends.rss_last_error，但友链列表暂时还没有展示详细错误字段，这个已记录为 deferred 项。

---
  8. 最终验收结论可以这样判断

  如果下面都满足，就可以认为 Batch 4 人工验收通过：

  1. 后台友链可以配置 RSS 地址。
  2. RSS 页面点击“刷新RSS”能抓取真实外部 RSS/Atom。
  3. 抓到的文章能显示在 RSS 列表。
  4. 刷新页面后文章仍存在。
  5. 标记已读后状态保持。
  6. 重复刷新不会生成重复文章。
  7. 错误 RSS 源会提示失败源数量，而不是假装成功。

  已知 deferred，不影响本轮验收：

  1. 定时刷新 RSS 尚未做，本轮只验收手动刷新。
  2. 友链管理页暂未展示每个源的 rss_status / rss_last_error 详细字段。
